# Agents Assemble Challenge: The Healthcare AI Endgame

**Hosted by:** Prompt Opinion

## 🎯 The Objective
Build interoperable healthcare agents at the intersection of **MCP (Model Context Protocol)**, **A2A (Agent-to-Agent)**, and **FHIR** standards. The goal is to solve "The Last Mile" of healthcare AI—converting raw intelligence into actionable deliverables.

## 💰 Prizes
- **Total Prize Pool:** $25,000 USD
- **Grand Prize:** $7,500 USD
- **Publishing:** Successful projects will be published in the Prompt Opinion Marketplace.

## 🛠️ What to Build
You must choose one of two technical paths for your submission:

### Option 1: Build a Superpower (Powered by MCP)
Build the "hammer." Create a custom MCP server that exposes a specific set of tools (which may or may not use patient data). By building an MCP, you forge a capability that any agent in the ecosystem can pick up and use.

### Option 2: Build an Agent (Powered by A2A)
Build a "superhero." Configure an intelligent Agent to handle complex healthcare workflows. Using Conversational Interoperability (COIN) and A2A standards, this agent will help other agents solve real problems. (Note: A2A development can be done natively within the Prompt Opinion platform without writing custom A2A protocol code).

## 📋 Submission Requirements
1. **Register:** Sign up on the Hackathon Website and create a free account at Prompt Opinion.
2. **Build:** Develop your MCP server or configure your A2A agent.
3. **Integrate (SHARP & FHIR):** Ensure your solution uses the **SHARP Extension Specs** to handle healthcare context (e.g., patient IDs and FHIR tokens). It is highly recommended to use data from a FHIR server in your solution.
4. **Publish:** Configure and publish your project to the Prompt Opinion Marketplace.
5. **Submit a Demo:** Provide a demonstration video (**under 3 minutes**) showing your project functioning within the Prompt Opinion platform.

How it Will Work
For this challenge, you must utilize the Prompt Opinion platform.

Register: Sign up on the Hackathon Website and create a free account at Prompt Opinion.

Build: Develop your MCP server or A2A agent using your own infrastructure and the reference implementations provided. You can also configure an A2A agent directly on the platform. Check out the overview video

Integrate: Ensure your solution uses the SHARP Extension Specs to handle healthcare context like patient IDs and FHIR tokens. While not required, it is highly recommended that you use data from a FHIR server in your solution.

Publish: Configure and publish your project to the Prompt Opinion Marketplace so it can be discovered and invoked within the platform.

Submit: Provide a demonstration video (under 3 minutes) showing your project functioning within the Prompt Opinion platform.


Why Prompt Opinion?
Prompt Opinion solves the "plumbing" problems that typically consume most of the engineering effort in healthcare AI.

Standards Built-In: The platform is natively built on MCP and A2A, ensuring your tools work across the entire ecosystem without custom glue code.

SHARP Context Propagation: The platform bridges EHR session credentials directly into SHARP context, so you don't have to invent bespoke token-handling solutions.

No Vendor Lock-In: Using open standards means any compliant tool or agent works, allowing for easy upgrades or swaps.

Agent Composition: Organizations can compose multiple specialist agents from the registry into a single workspace, allowing them to work together as one.
